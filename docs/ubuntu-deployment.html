准备域名和网络

先把域名指向服务器，否则 Caddy 无法签发公网证书。

  • 在 DNS 控制台创建 A 记录：mcp.example.com → 服务器公网 IPv4。
  • 只有在服务器确实配置了公网 IPv6 时才创建 AAAA 记录。
  • 云厂商安全组和 Ubuntu 防火墙都要允许 TCP 80、443。
  • 不要向公网开放 8788；应用只监听回环地址。
安装本指南使用的基础工具
sudo apt update
sudo apt install -y ca-certificates curl file
检查 DNS
getent ahosts mcp.example.com
注意：错误的 AAAA 记录很常见。它可能让浏览器或证书签发机构访问到错误的 IPv6 地址。

安装编译好的二进制

把用户目录中的构建产物复制到本机管理员安装目录。

核对架构
uname -m
file "$HOME/rs-mcp-lunar/target/release/rs-mcp-lunar"

两条命令显示的 CPU 架构应一致，然后安装：

安装到 /usr/local/bin
sudo install -o root -g root -m 0755 \
  "$HOME/rs-mcp-lunar/target/release/rs-mcp-lunar" \
  /usr/local/bin/rs-mcp-lunar

ls -lh /usr/local/bin/rs-mcp-lunar
为什么放这里：/usr/local/bin 用于管理员在系统包管理器之外安装的本地程序，也不会依赖你的 Home 目录权限。
2

创建 systemd 系统服务

使用无登录权限的专用用户运行，并在崩溃后自动重启。

创建服务用户
getent passwd lunar-mcp >/dev/null || sudo useradd \
  --system --user-group --no-create-home \
  --home-dir /nonexistent --shell /usr/sbin/nologin lunar-mcp

如果提示用户已经存在，可以直接继续。接着创建 unit：

/etc/systemd/system/rs-mcp-lunar.service
sudo tee /etc/systemd/system/rs-mcp-lunar.service >/dev/null <<'EOF'
[Unit]
Description=BaZi and Zi Wei Dou Shu MCP Server
After=network.target

[Service]
Type=exec
User=lunar-mcp
Group=lunar-mcp
ExecStart=/usr/local/bin/rs-mcp-lunar
Environment="LUNAR_MCP_ADDR=127.0.0.1:8788"
Environment="RUST_LOG=info"
Restart=on-failure
RestartSec=3s
KillSignal=SIGINT
TimeoutStopSec=15s
UMask=0077

NoNewPrivileges=true
PrivateTmp=true
PrivateDevices=true
ProtectSystem=strict
ProtectHome=true

[Install]
WantedBy=multi-user.target
EOF
校验并启动
sudo systemd-analyze verify /etc/systemd/system/rs-mcp-lunar.service
sudo systemctl daemon-reload
sudo systemctl enable --now rs-mcp-lunar.service
systemctl status rs-mcp-lunar.service --no-pager
sudo ss -lntp | grep ':8788'

状态应显示 active (running)，监听结果应只有 127.0.0.1:8788。

5

放行 80 和 443

先保留 SSH，再启用 Ubuntu 默认的 UFW 防火墙。

远程服务器务必先放行实际 SSH 端口。下面的 OpenSSH 规则适用于默认 22 端口；如果你改过端口，请先放行对应的 TCP 端口，否则启用 UFW 可能锁死连接。
UFW
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status verbose

如果云厂商还有安全组，也需要在那里开放 TCP 80、443。无需开放 8788。

6

配置域名和自动 HTTPS

Caddy 看到公网域名后，会自动监听 443、申请证书并把 80 重定向到 HTTPS。

先备份，再编辑
sudo cp /etc/caddy/Caddyfile /etc/caddy/Caddyfile.bak
sudo nano /etc/caddy/Caddyfile

如果这是专用服务器，可用下面内容替换文件；如果已有其他站点，只追加这个站点块：

Caddyfile
mcp.example.com {
  reverse_proxy 127.0.0.1:8788
}
SSE 无需额外配置：Caddy 检测到 Content-Type: text/event-stream 时会立即刷新响应，不需要手动设置 flush_interval。
格式化、校验、无中断重载
sudo caddy fmt --overwrite /etc/caddy/Caddyfile
sudo caddy validate --config /etc/caddy/Caddyfile --adapter caddyfile && \
  sudo systemctl reload caddy
systemctl status caddy --no-pager
7

验证公网 HTTPS

证书首次签发通常只需要几秒；DNS 尚未生效时会更久或失败。

确认 HTTP 自动跳转 HTTPS
curl -I http://mcp.example.com/

应看到 308 和指向 HTTPS 的 Location 响应头。

HTTPS 信息页
curl -fsS https://mcp.example.com/
公网 MCP initialize
curl --fail-with-body --no-buffer https://mcp.example.com/lunar \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  --data '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"deploy-check","version":"1.0.0"}}}'

最后一条命令应返回 SSE，并在 serverInfo 中显示 Lunar Calendar MCP。

HTTPS 不等于鉴权。该 MCP 服务没有内建登录机制。若不希望任何人调用，请在 Caddy 前增加你所使用的 MCP 客户端支持的鉴权方案，或通过 VPN / IP 白名单限制访问。

后续更新二进制

先安装到临时文件，再原子替换并重启服务。

重新构建与发布
cd "$HOME/rs-mcp-lunar"
cargo build --release --locked

sudo install -o root -g root -m 0755 \
  target/release/rs-mcp-lunar \
  /usr/local/bin/rs-mcp-lunar.new

sudo mv /usr/local/bin/rs-mcp-lunar.new /usr/local/bin/rs-mcp-lunar
sudo systemctl restart rs-mcp-lunar
systemctl status rs-mcp-lunar --no-pager

Caddy 配置没有变化，因此更新应用时不需要重启 Caddy。

故障排查

按现象快速定位，先看日志，再检查 DNS 和端口。

systemd 服务启动失败
应用日志
sudo journalctl -u rs-mcp-lunar -n 100 --no-pager
sudo journalctl -u rs-mcp-lunar -f
常见原因：二进制架构不匹配、文件不可执行、8788 已被其他进程占用。
Caddy 返回 502 Bad Gateway
这表示 Caddy 无法连接应用。先运行：
后端检查
systemctl is-active rs-mcp-lunar
curl -v http://127.0.0.1:8788/
HTTPS 证书申请失败
确认 A/AAAA 记录、云安全组、UFW、服务器时间和 80/443 端口，然后查看：
Caddy 日志
sudo journalctl -u caddy -n 100 --no-pager
检查当前监听端口
监听状态
sudo ss -ltnp | grep -E ':(80|443|8788)\b'
预期：Caddy 监听 80/443，rs-mcp-lunar 只监听 127.0.0.1:8788。
